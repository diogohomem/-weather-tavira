Informações e Instruções de Funcionamento

O Weather Tavira é uma aplicação web meteorológica que apresenta as condições atuais e a previsão para 7 dias em Tavira (Algarve, Portugal). Está disponível em duas modalidades: versão online e versão local.

✅ Versão Online — Recomendada
Acesso imediato via browser, sem instalar nada.
URL: https://web-production-9aff6.up.railway.app

Páginas disponíveis

PáginaURL (online)Descrição
Widget Principal/Mostra o widget de embed com temperatura do ar, temperatura da água e previsão 7 dias
Dashboard Completo/dashboard.htmlPainel detalhado: temperatura, humidade, vento, pressão, previsão horária e semanal. Permite pesquisar qualquer cidade do mundo
Widget Embed/widget.htmlVersão standalone do widget para incorporar noutros sites via iframe

Como usar o Widget em outro site (Embed)

Na página principal, clica em "Copiar código" — o código abaixo é copiado para a área de transferência e pode ser colado em qualquer site HTML:

<iframe
  src="https://web-production-9aff6.up.railway.app/widget.html"
  width="320" height="520"
  style="border:none;border-radius:16px"
  loading="lazy"></iframe>

Iniciar a versão local (requer Node.js instalado)

1
Abrir o Terminal na pasta do projeto Mac: clicar com o botão direito na pasta "Weather Tavira" → "Novo Terminal na Pasta"
2
Instalar dependências (apenas uma vez) Escrever no Terminal: npm install
3
Iniciar o servidor Escrever: npm run dev — o browser abre automaticamente em http://localhost:3000
4
Manter o Terminal aberto O servidor fica ativo enquanto o Terminal estiver em execução.
⚠️ Nota importante — versão local
O ficheiro .env na raiz do projeto contém a chave da API. Deve estar presente para o servidor funcionar. Se não existir, o servidor encerra automaticamente com uma mensagem de erro clara.

Mecanismos de proteção implementados

  • Timeout de 8 segundos em todas as chamadas a APIs externas — o servidor não fica "pendurado" indefinidamente
  • Timeout de 10 segundos no cliente — mostra mensagem clara se o servidor demorar
  • Verificação de resposta antes de processar dados — erros da API são comunicados claramente
  • Temperatura da água "best effort" — se a API Marine falhar, o resto da app continua a funcionar normalmente
  • Auto-refresh automático — widget atualiza de 15 em 15 minutos; dashboard de 10 em 10 minutos

Nota Executiva — Evolução e Correções

Ponto de partida

O projeto foi iniciado como um widget meteorológico simples para Tavira. A arquitetura escolhida — servidor Node.js local como proxy — permite proteger a chave API e fazer chamadas a múltiplas fontes de dados externas sem expor credenciais no browser.

Evolução do projeto

FaseO que foi feito
Fase 1 — BaseServidor Express com endpoints /api/weather e /api/widget-data; widget com temperatura do ar e do mar; dashboard com pesquisa de cidades
Fase 2 — EstabilidadeIdentificação e correção de bugs críticos (ver tabela abaixo); implementação de mecanismos de timeout e proteção de erros
Fase 3 — DeployPublicação online no Railway com variável de ambiente segura; repositório GitHub; documentação

Bugs identificados e correções aplicadas

BugImpactoCorreçãoEstado
Servidor sem timeout nas chamadas às APIs externas Servidor podia ficar "pendurado" indefinidamente se a API não respondesse, tornando a aplicação inacessível Implementado AbortController com timeout de 8 segundos em todas as chamadas externas. Erros de timeout distinguidos de outros erros ✔ Corrigido
Data do dashboard no fuso horário local do computador Utilizadores fora de Portugal (UTC+1) viam o dia errado — ex: mostrava "Quarta" quando era "Quinta" Substituído .getDay()/.getDate()/.getMonth() por .getUTCDay()/.getUTCDate()/.getUTCMonth() após ajuste correto do offset ✔ Corrigido
Ícones meteorológicos errados para névoa/neblina Condições de nevoeiro (códigos OpenWeather 700-799) mostravam o ícone de nuvens em vez do correto Reordenamento da função codeToIcon() — o bloco 700-799 estava após o bloco 803+ e nunca era alcançado (código "unreachable") ✔ Corrigido
Agrupamento de dias da previsão semanal pelo fuso local À meia-noite, o mesmo dia podia ser dividido em dois grupos diferentes, gerando dias duplicados ou ausentes na previsão Agrupamento alterado para usar getUTCFullYear/Month/Date — consistente com o timezone da localização meteorológica ✔ Corrigido
Dia da semana no widget com timezone local Função fmtDay() interpretava datas no fuso horário do sistema. Utilizadores fora de Portugal podiam ver o dia errado Adicionado sufixo Z ao parsing da data (T12:00:00Z) para forçar interpretação UTC ✔ Corrigido
Sem validação da resposta OpenWeather antes de parsear Se a API devolvesse erro 401/429, o servidor tentava aceder a current.main.temp e crashava internamente com TypeError Adicionada verificação .ok antes de aceder aos dados, com mensagem de erro clara em português ✔ Corrigido
Código de embed com URL placeholder O botão "Copiar código" copiava SEU-DOMINIO em vez do URL real — código copiado não funcionava URL real do Railway inserido; adicionado fallback com prompt() para browsers que bloqueiam a clipboard API ✔ Corrigido

Estado atual

✅ Programa estável
Todos os bugs críticos identificados foram corrigidos. O servidor tem proteção de timeout em todos os pontos de falha. Os dados apresentados são fiáveis. A versão online está em produção no Railway e acessível a qualquer utilizador com browser e internet.

Limitações conhecidas (não críticas)

  • Plano Railway gratuito — o servidor "adormece" após inatividade; a primeira visita após inatividade pode demorar 10-15 segundos
  • Chave API OpenWeather gratuita — limite de 1.000 chamadas/dia; para uso académico é mais do que suficiente
  • Temperatura da água — depende da disponibilidade da API Open-Meteo Marine; pode mostrar "—" em determinados períodos
  • Previsão semanal — baseia-se na API gratuita Open-Meteo (7 dias); o dashboard usa a OpenWeather que limita a 5 dias no plano gratuito

Possíveis melhorias futuras

  • Cache no servidor para reduzir chamadas às APIs e acelerar respostas
  • Suporte a mais cidades/localizações no widget (atualmente fixo em Tavira)
  • Notificações de condições extremas (tempestade, calor excessivo)
  • Gráfico de temperatura das últimas 24h
